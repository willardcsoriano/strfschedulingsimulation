  • Job (PID): Unique identifier.
  • Arrival Time (AT): Time job arrives (1-7).
  • Burst Time (BT): Total execution time required (1-10).
  • Completion Time (CT): $\text{AT} + \text{TAT}$.
  • Turnaround Time (TAT): $\text{CT} - \text{AT}$. Total time in system.
  • Waiting Time (WT): $\text{TAT} - \text{BT}$. Time spent idle in the queue.

üß† SRTF Algorithm Logic

The **Shortest-Remaining-Time-First (SRTF)** algorithm is a **preemptive** scheduler. It always executes the job currently in the ready queue with the shortest time *remaining* to completion.

  1. Check for new arrivals at the current time $t$.
  2. Select the job with the minimum $\text{Remaining Time}$ from all arrived jobs.
  3. Execute for 1 time unit.
  4. If a new job arrives with a shorter $\text{Remaining Time}$, the CPU **preempts** the current job and switches.
