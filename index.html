Simulation Objective: SRTF Analysis

This tool simulates 100 randomly generated jobs to demonstrate the efficiency of the Shortest-Remaining-Time-First (SRTF) scheduling algorithm.

How SRTF Works (Preemption):

SRTF is a preemptive CPU scheduling algorithm. It always runs the job that needs the least amount of time remaining to finish. If a shorter job arrives, it immediately interrupts (preempts) the running job.

Job Parameters & Goal Metrics:

  • Burst Time (BT): Randomly set between 1 and 10.
  • Arrival Time (AT): Randomly set between 1 and 7.
  • Goal: Minimize Average Waiting Time (WT) and Turnaround Time (TAT).

Real-Life Application üåé:

This scheduling concept is critical in Real-Time Operating Systems (RTOS) used in applications like avionics, industrial control, and robotics. By prioritizing the shortest task remaining, SRTF helps guarantee that time-critical tasks (like sensor readings) are executed with the lowest possible latency.

üìä Column Definitions

  • Job (PID): Unique identifier.
  • Arrival Time (AT): Time job arrives (1-7).
  • Burst Time (BT): Total execution time required (1-10).
  • Completion Time (CT): $\text{AT} + \text{TAT}$.
  • Turnaround Time (TAT): $\text{CT} - \text{AT}$. Total time in system.
  • Waiting Time (WT): $\text{TAT} - \text{BT}$. Time spent idle in the queue.

üß† SRTF Algorithm Logic

The Shortest-Remaining-Time-First (SRTF) algorithm is a preemptive scheduler. It always executes the job currently in the ready queue with the shortest time *remaining* to completion.

  1. Check for new arrivals at the current time $t$.
  2. Select the job with the minimum $\text{Remaining Time}$ from all arrived jobs.
  3. Execute for 1 time unit.
  4. If a new job arrives with a shorter $\text{Remaining Time}$, the CPU preempts the current job and switches.
